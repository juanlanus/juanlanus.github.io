Associations builder

To mimic the logical flow of the user's mental model, the association selection has to be between the "From" and the "To" trees. The users think "John loves Jane" and not "John and Jane, love" in Polish notation. The Polish notation ig easy to implement in programming, but it requires a paradigm shift from the users.

The proposed UI looks more or less like this:

Before building an association all its three components are highligted (as in the example, shown with a gray border and a light gray background). This allows the user to check the association that's about to be built with ease, i.e., reduces human error.

In addition to highlighting the elements of the association, the user sees it expressed in English text, next to the "Associate" button. The association description grows as the user selects the components, like "John Doe ...", then "John Doe works for ..." after selecting the association type, etc... The "Associate" button is disabled until after all the components are selected.

In some cases the UI should allow the selection of tree nodes in the "From" side, that is, allow to associate (for example) all the members of an organizational group at once, like "London employees are payed by London Region".

In the case of multiple associations built in a single operation, it is mandatory to tell the user the number of items affected. The button should read "Associate 245 items" or so.

This arrangement allows the user to build associations using the keyboard. The Tab key has to stop inthe selected item of the "From" artifact, the "Associations" list, the "To" list, and the "Associate" button (that has to be a default button, i.e., connected to the Enter key.

The user should also be able to drag a "From" item over an "Associations" item and drop it on a "To" item to specify an association in a single swoop. Hovering an item in "Associations" with the loaded cursor has to select it.

Another way to build multiple associations using drag and drop is to drop one or many "From" items into an association item, and then drag the whole lot into the "To" part.

The grid labeled "The following asociations have been created" conveys some wishful thinking. In a not-so-small organisation there can be hundreds, if not thousands, of associations. Also, the names of the associated items could be rather long, like for example Government offices involving a function and a jurisdiction.

Users that are building many associations, and need to check if one is already built (among hundreds) need a means to quickly filter the list to make it shorter and be able to scan it. In these cases a quick "contasins" filter is the most effective solution.

Similarly, if the user has to pick items from within rather big organisational structures, the corresponding trees ("From" and "To") need to be filter-able too.
