Prefer radio buttons over drop-downs

An array of (a few) radio buttons is preferable over a dropdown offering the same options.

The reason is that radios require only one click while dropdowns require multiple clicks.

Another advantage of radios is that all the options are visible at once, while in a dropdown all options but one are concealed and the user needs to open it to be aware of the choices.

It is possible to use radios when the options are not many, like up to 8 for example. When the options are many, a dropdown is the choice.

Many times dropdowns are preferred because it is easier to configure them programmatically, albeit it is also possible to do so with radio buttons.

On the downside, the only option that a dropdown lets see is the selected one, while in a traditional radio buttons group the user has to scan the array in order to find the selected option. This can be easily solved by highlighting the label of the selected one, maybe using a mere CSS3 rule like input[type="radio"]:checked+label {border:2px solid rgb(128,128,128);}

There is a Telerik example of radio buttons that highlight the selected one beyond the mere inner circle.

When setting the radio buttons in a row (as in the example above) ensure that the label is close to the related button, as in the example where there is some white space after each label. Use a proportional unit like ems, to mantain the separation in case the user enlarges the font size.
