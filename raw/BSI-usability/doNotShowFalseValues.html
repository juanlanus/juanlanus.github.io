Do not show false values

In grid columns depicting a binary value, do not show the false values but leave them blank. In the example below (from Manage Association Types) every row has a mark, either a checkmark or a "x".

This makes more difficult to scan the column looking for a particular value. Obviously, in this small example it is not an issue, but in real life long lists it is.

In the example below, there are only checkmarks, the "x"s representing the false value are blank instead. This way the differences stand out immediately.

The same can be said about numeric values. Omitting the zeros usually leads to much easier to reac columns.
