BSI Usability

Use coherent binary labeling

The example below has a column, labeled "Enable", that contains binary yes or no values.

The issue is that "Enable" names an action, making such a label appropriate for action controls like buttons.

A label for a yes/no column has to be something that could be formulated as a question, in this case "Enabled?".

The question mark can be spared, leaving "Enabled" as the correct label.

These details have a distracting impact on the user, who has to stop for a fraction of a second to check if there is something that she got wrong, or is the computer who is wrong. A good UI does not have any glitches of this kind, and the user´s mind can flow straight to acieving her goals.

Data has to be labeled with names, actions (buttons) with verbs. In particular, binary data has to be labeled as assertions, like "has property X" which can be true, false or sometimes not known.
