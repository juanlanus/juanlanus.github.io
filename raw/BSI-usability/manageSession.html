Manage session

Session expiration has to be managed globally.

Attempting to change a grid's number of rows from within an expired session returned an error like a failed XHR.

It might well be already fixed, but it raised the question of what to do with expired sessions in order to enhance the user experience, be unobstrusive, and have to develop a simple, single, and elegant solution.

An expired session might be a user that departed from the office until Monday, or a user that had a phone call from her boss and is eager to continue her work.

The most valued solution is to lock the window covering it with a gray shade containing a login form at its center. This is how Windows behaves, but it is a good way to do it.

The session data in the server has to be saved, so it can be recovered in case the user attempted to revive the session. To detect it, the reconnect login form has to have a token to be used to relate the saved session with the reconnect action.

Should the same user attempt to reconnect without the token, then she would be presented a completely new session and the saved data could be erades.
