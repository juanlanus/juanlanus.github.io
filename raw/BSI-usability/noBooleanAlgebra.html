Users don't understand Boolean algebra

It is clear that users don't understand Boolean algebra, for all but IT people (like me, it took me years to realize this). This is the finding of numerous user tests regarding search technologies, a significant function of the web.

This is so because speach implicit subjects trump the Boolean logic. For example, "the records belonging to London and Lancaster" in Boolean should return zero records while in speach it expected to return "the records belonging to London or Lancaster".

"People interpret the word AND to mean either conjunction or disjunction depending on context and the scope they attribute to the word NOT depends on whether the subsequent operator is AND or OR. Therefore, relying on these words and symbols for query formulation will result in poor usability."

However, Boolean queries are valuable, and Entropy 5 users might stand out from the crowd by a lot. Thus, although there is no need to replace the Boolean query builder it might be of some value to add a litle help in case somebody is dubious about the effect of a query, or ran one and got unexpected results.

In the example above, the query builder has its usual fields, and a text explaining, in plain English, what will the query do.

Also, the query builder must have a really good two-level manual page, with a brief introduction, and a detailed tutorial to be spared by Boolean savvy users.
