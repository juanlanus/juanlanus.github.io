Dialogues should not contain information which is irrelevant or rarely needed. Every extra unit of information in a dialogue competes with the relevant units of information and diminishes their relative visibility.

The Windows 7 file overwrite dialog is a counter-example of this rule.

How could we re-design it in order to make it more usable?

Besides having too busy an UI, this dialog has a number of issues:
  • The "Copy File" text in the dialog title is not informative. This is a premium location and should be more informative. IMO users have developed dialog title blindness due to the lameness of dialog titles, just space holders. It can inform the user about what's happening, like in "Copy newer version of existing file" to help then take the right decision in a split second.
  • It says "There is already a file with the same name in this location" but the mentioned location should show as part of the message.
  • The word "location" in "There is already a file with the same name in this location" is allien. We seldom use this word, it should be "folder" or "directory".
  • The "Click the file you want to keep" instructions are misleading. Where is "the file I want to keep"? It should be something like "Choose one of the options below".
  • The "Copy and Replace" label mentions two operations: copy and replace. To the user it's only one, no matter if the program internally first copies the incoming file and after does the replacement. To convey a sense of singleness, "Replace" would be a better fit.
  • The first and second options have file information, whilst the third doesn't. This is because tha data is already depicted in the previous options, but it makes this option different, a second calss option, conveying a message like don't click me, the other choices are right.

    File information doesn't belong to the options but instead is global information, and should not be tied to them.

  • (newer) is good but last: the user didn't read this much
